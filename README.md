# Volume Data Visualizer for Google Earth (VDVGE)

*[Japanese version is available here.](README.ja.md)*

## Overview

Volume Data Visualizer for Google Earth (VDVGE) is a software designed to visualize 3D data used in [GrADS](https://cola.gmu.edu/grads/) (a plotting tool for earth science data) and to export the visualization results in a format compatible with [Google Earth](https://earth.google.com/).

Through simple GUI-based operations, VDVGE enables volume-rendering-like representations on Google Earth Pro using KML and COLLADA files.

Developed using [Qt](https://www.qt.io/), a multi-platform GUI toolkit, VDVGE shares a common source code and runs seamlessly on Microsoft Windows, Mac OS X, and Linux. Currently, this software is mainly dedicated to atmospheric data, but support for oceanic data is planned for future updates.

## Sample Data

The sample data provided in the release page is generated by the general circulation model [AFES](https://gitlab.com/aosg_public/afes) (AGCM for Earth Simulator). It contains a single physical parameter (cloud water content) that has been cropped and thinned out around the ocean near Japan.

## Input Data Restrictions

The GrADS descriptor file for VDVGE has the following restrictions:

* The file specified by the `DSET` keyword must be an unformatted FORTRAN binary (single-precision real numbers). It must be an actual existing file, as the template feature (`TEMPLATE` option) is not supported.
* The GrADS control file can contain only a single scalar field (i.e., only one parameter in the `VARS` keyword). If the descriptor file contains multiple physical quantities, please separate them into individual files for each quantity.
* The unit for `ZDEF` must be specified in actual altitude (meters or kilometers). If the original data uses other units such as atmospheric pressure, pre-conversion is required.
* Most other unsupported commands and options in the control file will be ignored.

## Requirements

* **Qt**: Required to compile VDVGE. Please refer to the manual for compilation instructions.
* **FFmpeg**: Required if you use the movie export function.
* **Topography and Coastline Data**: ETOPO1(etopo1_ice_g_i2.bin), ETOPO2(ETOPO2v2g_i2_LSB.bin), or ETOPO5(ETOPO5.DOS) is required for the topography drawing function, and GEODAS Coastline is required for the coastline drawing function.
* **Graphics Hardware**: Hardware that supports OpenGL 1.2 or higher is required, as VDVGE uses the 3D texture feature of OpenGL.
